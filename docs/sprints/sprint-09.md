# Sprint 9: The Blog & Admin Email Marketing Engine

## Sprint Goal
Deploy the organic pull-marketing engine by rendering SEO-optimized blog articles. Establish the direct retention channel by building an Admin UI to draft and broadcast email campaigns (newsletters, drop announcements) to registered users via a dynamically configured Email API provider (e.g., Resend or SendGrid).

> **🛡️ PRO-TIP / SECURITY DIRECTIVE FOR THE ORCHESTRATOR:** > When the Frontend Specialist builds the blog article rendering component (`/blog/[slug]`), they MUST use a safe HTML sanitizer (like `dompurify` or `isomorphic-dompurify`) or a strict parser like `next-mdx-remote`. Never use dangerouslySetInnerHTML with raw database strings without sanitizing first, even if the content originates from our own secure Admin CMS.

---



## Phase 1: Feature Execution (70% Effort)

### Backend Requirements (Delegate to @backend-specialist)
1. **Dynamic Sitemap (`app/sitemap.ts`):** Write a Next.js dynamic sitemap generator. It must query Supabase for all active `Products` and published `BlogPosts`, returning their absolute URLs and `updated_at` timestamps for Google indexing.
2. **Email Provider Integration:** Write a secure Edge Function (`/api/admin/email/broadcast`). 
   - It must dynamically fetch the `EMAIL_API_KEY` (e.g., Resend) from the `StoreSettings` table.
   - It must query the `Users` table to fetch all registered customer email addresses (or those who opted-in to marketing).
   - Use the provider's SDK/REST API to dispatch the drafted HTML email payload to the user list.
3. **API Contract:** Update `docs/api_contract.md` with the email broadcast payload and any new SEO metadata structures required from the database.

### Frontend Requirements (Delegate to @frontend-specialist)
1. **The Blog Storefront (`/blog` & `/blog/[slug]`):** - Build the main blog index displaying article cards (Title, Excerpt, Cover Image, Publish Date).
   - Build the individual article page. Ensure the rich text generated by the CMS renders flawlessly with typography that matches the "Old Money" aesthetic (e.g., elegant drop caps, wide reading margins). **Apply strict HTML sanitization here.**
2. **SEO Metadata:** Utilize Next.js `generateMetadata()` on the product and blog pages to dynamically inject `<title>`, `<meta name="description">`, and OpenGraph/Twitter Card tags using the database content.
3. **Admin Email Campaigns UI:** Add the "Email Campaigns" module to the `/admin` dashboard.
   - Build a form to draft a Subject Line and Email Body (using the existing rich-text editor component).
   - Add a "Send Test Email" button (sends only to the admin's email).
   - Add a "Broadcast to All Users" button with a strict confirmation modal to prevent accidental misfires.

---

## Phase 2: Hardening & Tech Debt (30% Effort)

### QA & Integration (Delegate to @qa-integration)
1. **SEO Audit:** Inspect the DOM of a rendered `/blog/[slug]` page and a `/shop/[id]` page. Verify that all Next.js generated OpenGraph meta tags are present and accurately reflect the database content. Ensure the `sitemap.xml` returns a valid 200 response with correct URLs.
2. **Email Security Audit:** Attempt to call the `/api/admin/email/broadcast` endpoint as an unauthenticated user or standard customer. Verify it strictly returns a 403 Forbidden error.
3. **XSS Vulnerability Check:** Attempt to save a blog post containing a malicious `<script>alert('hacked')</script>` tag. Verify that the frontend sanitizer strips the script and prevents it from executing when the article is viewed.

---

## Definition of Done (Orchestrator Review)
- [ ] Blog posts render beautifully on the frontend and are strictly sanitized against XSS attacks.
- [ ] Dynamic `sitemap.xml` and OpenGraph meta tags generate correctly for SEO.
- [ ] Admins can draft and successfully dispatch test emails from the dashboard.
- [ ] Email API keys are fetched dynamically and never exposed.
- [ ] The Orchestrator has updated `docs/MEMORY.md` with decisions regarding HTML sanitization and SEO strategies.
